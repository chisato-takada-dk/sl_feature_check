2. コード再利用の例外 [hide]

次のセクションでは、関数のパッケージ化設定が以下の場合に、非バーチャル サブシステムが以下のようになる原因となる例外の詳細を示します。

  • [自動] の場合に、インライン コード セグメントになる
  • [自動] の場合に、引数をもたない、再利用できない関数になる
  • [再利用可能な関数] の場合に、引数をもたない、再利用できない関数になる
メモ:このセクションでは、機能プロパティ (次元、データ型、作業ベクトル、パラメーターなど) の違いにより再利用されなかった、[自動] とマークされたグラフィカルに同一な非バーチャル サブシステムについては説明しません。そのような場合は、モデル内またはインライン生成コード内のサブシステムの機能属性の違いを検査することで、再利用できない理由を同定できます。
